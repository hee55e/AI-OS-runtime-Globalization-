# 기타 핸들러

> 코드 동기 기준: 2026-02-03
> 원칙: 문서보다 코드가 우선입니다.

## 현재 구현
- 기타 핸들러로 `llm`, `subagent`, `tcp`, `udp`, `webrtc`, `computer_use`, `direct` 등이 존재합니다.
- 각 핸들러는 UTCP provider_type과 1:1 또는 alias 관계로 레지스트리에 등록됩니다.
- 플러그인 경로 추가 시 manifest auto-discovery로 확장 가능합니다.

## 프론트엔드 연결
- PluginsView에서 핸들러/다이렉트/필터 플러그인 상태를 조회합니다.

## 백엔드 연결
- 핸들러 루트: `kait_os/tools/handlers/`
- 플러그인 API: `kait_os/routers/system/plugins.py`
- 핸들러 레지스트리 아키텍처: `docs/HANDLER_REGISTRY_ARCHITECTURE.md`
