# KAIT

> 코드 동기 기준: 2026-02-03
> 원칙: 문서보다 코드가 우선입니다.

## 현재 구현
- KAIT는 AI Instance의 그래프 실행을 중심으로 동작하는 오케스트레이션 런타임입니다.
- 요청 흐름은 `conversation -> orchestrator -> graph executor -> processor/tool` 순서로 실행됩니다.
- 도구는 UTCP 매뉴얼(JSON) 기반으로 로드되며 실행은 핸들러 레지스트리 디스패처가 담당합니다.
- 멀티모달은 UCF 규격을 기준으로 처리되고 필요 시 JIT 해석(`MultimodalHandler`)이 적용됩니다.

## 프론트엔드 연결
- 스튜디오 화면은 단일 앱 셸이며 사이드바/뷰포트/패널 조합으로 동작합니다.
- 상태 동기화는 SSE(`/api/state-events`) 이벤트를 EventBus가 받아 스토어에 반영합니다.

## 백엔드 연결
- 엔트리/라우터 등록: `kait_os/app.py`
- 대화 스트림 엔드포인트: `kait_os/routers/conversation/conversation.py`
- 그래프 조율기: `kait_os/core/orchestration/graph/`
- UTCP 실행기: `kait_os/tools/tool_executor.py`
