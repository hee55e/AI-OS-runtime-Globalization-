# 조립하기 (노드, 엣지, 프로세서)

> 코드 동기 기준: 2026-02-03
> 원칙: 문서보다 코드가 우선입니다.

## 현재 구현
- 그래프는 `AIInstance.cognitive_pipeline`에 저장되고 실행 시 GraphExecutor가 해석합니다.
- 노드 타입은 manifest 기반(12종), 엣지 타입은 manifest 기반(9종)으로 로드됩니다.
- 프로세서는 재사용 가능한 ProcessorUnit이며 노드에서 `processor_id`로 참조합니다.
- 표현식 `${...}`/`{{...}}`와 `:multimodal` 모디파이어를 지원합니다.

## 프론트엔드 연결
- 그래프 편집기: `GraphEditor.tsx`, `GraphCanvas.tsx`
- 노드 팔레트: `NodePalettePanel.tsx`
- 속성 패널: `PropertiesPanel.tsx`

## 백엔드 연결
- 노드 manifest: `kait_os/core/orchestration/graph/nodes/*/manifest.json`
- 엣지 manifest: `kait_os/core/orchestration/graph/edges/types/*/manifest.json`
- 표현식 해석: `kait_os/core/common/utils/expression_resolver.py`
- 파이프라인 검증: `kait_os/services/execution/pipeline_validator.py`
