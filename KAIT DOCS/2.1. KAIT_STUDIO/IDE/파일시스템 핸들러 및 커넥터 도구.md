# 파일시스템 핸들러 및 커넥터 도구

> 코드 동기 기준: 2026-02-03
> 원칙: 문서보다 코드가 우선입니다.

## 현재 구현
- 파일시스템 핵심 UTCP 도구는 read/write/edit/create_directory/delete 및 검색 계열입니다.
- 커넥터 설정에서 허용 경로와 위험 동작 정책(delete/rmtree/overwrite)을 사용자 단위로 저장합니다.
- 로컬 파일 API는 보안 경로 검증으로 접근 범위를 제한합니다.

## 프론트엔드 연결
- 파일시스템 정책 UI: `components/connectors/FileSystemConnectorPanel.tsx`
- 파일 탐색 UI: `components/sidebar/FileExplorerView.tsx`

## 백엔드 연결
- 핸들러 구현: `kait_os/tools/handlers/filesystem/`
- UTCP 도구: `kait_os/utcp/categories/filesystem/*.json`
- 로컬 파일 브라우저: `kait_os/routers/data/file_browser.py`
