# 도구 제작 및 바인딩

> 코드 동기 기준: 2026-02-03
> 원칙: 문서보다 코드가 우선입니다.

## 현재 구현
- 시스템 도구는 UTCP 매뉴얼(`utcp/categories/**/*.json`)에서 로드됩니다.
- 사용자 도구는 `user_tools` 테이블에 저장되며 Tool Builder API로 CRUD/Test를 수행합니다.
- LLM에서 도구 사용은 ProcessorUnit.allowed_tools -> ToolBindingHandler 변환 경로를 탑니다.
- 도구 실행은 Permission -> Policy -> Schema 3단계 검증 후 핸들러 디스패치가 수행합니다.

## 프론트엔드 연결
- Tool Builder 모달/에디터: Sidebar의 도구 생성 액션
- 사용자 도구 목록/테스트: API client 기반 UI

## 백엔드 연결
- 시스템 도구 목록: `kait_os/routers/tools/utcp_tools.py`
- 사용자 도구 CRUD: `kait_os/routers/tools/user_tools.py`
- 바인딩 변환: `kait_os/core/execution/tool_binding_handler.py`
- 실행기: `kait_os/tools/tool_executor.py`
