# 트리거

> 코드 동기 기준: 2026-02-03
> 원칙: 문서보다 코드가 우선입니다.

## 현재 구현
- 트리거 정의는 AI 파이프라인(`cognitive_pipeline`) 내부에서 파싱됩니다.
- TriggerPanel은 `api | schedule | event_bus | webhook` 타입 편집 UI를 제공합니다.
- 백엔드는 목록/상태/수동 발동/리로드/이벤트 emit/스케줄러 상태 API를 제공합니다.

## 프론트엔드 연결
- 트리거 패널: `components/triggers/TriggerPanel.tsx`

## 백엔드 연결
- 라우터: `kait_os/routers/execution/triggers.py`
- 로더/실행 서비스: `kait_os/services/events/trigger_loader_service.py`, `trigger_execution_service.py`
- 트리거 파서: `kait_os/schemas/cognitive_architecture.py`
